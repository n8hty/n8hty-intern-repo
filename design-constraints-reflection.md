# Handling Design Constraints & Technical Feasibility

## If a developer tells you a design is too complex to implement, how would you respond?
I would ask which specific part is difficult to build instead of immediately defending the design. For example, if a developer in says that animated transitions between focus
sessions may cause lag on lower-end devices, I would suggest reducing the animation length or using simpler fades so the interface still feels smooth without affecting 
performance.

## What are some non-visual factors (e.g., performance, accessibility, load time) that can impact a UX design?
Even a clean interface can feel frustrating if the app is slow or difficult to navigate. In a productivity app like , delays when starting a focus session or poor keyboard 
navigation could interrupt a user’s concentration, especially for neurodivergent users who rely on fast and predictable interactions.

## How can UX designers help find middle-ground solutions between ideal design and technical feasibility?
UX designers can break features into smaller priorities instead of treating every detail as required. For example, if developers cannot build a fully customizable dashboard 
before launch, the team could first release a simpler layout with only the most important widgets, then add advanced customization later based on feedback and development 
capacity.
