# How UX Works with Other Teams

# Reflection

## How can UX ensure a smooth handoff between design and development?
A smooth handoff requires clear communication and organized files. During my internship, I learned the importance of labeling components and adding notes in Figma 
so developers can understand spacing, interactions, and user flows without confusion. This helps avoid unnecessary revisions.

## What challenges might arise when design, dev, and product teams have conflicting priorities?
Conflicting priorities can lead to tradeoffs between features and usability. For example, I can imagine the product team wanting more customization options in 
Focus Bear, while UX aims to keep the interface simple for neurodivergent users. In this situation, I would suggest releasing essential features first and 
improving them based on feedback.

## If a developer modifies a UX design due to technical limitations, how should a UX designer respond?
A UX designer should collaborate with developers to find alternatives rather than insisting on the original design. For example, if complex animations slow down 
the app, I would simplify them while keeping the experience clear and supportive. I have learned that flexibility and communication are important for balancing 
user needs and technical constraints.
