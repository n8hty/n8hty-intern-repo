# How to Conduct User Interviews & Usability Testing

## If a user struggles with a feature during testing, how do you know if it’s a UX problem or a learning curve issue?
If multiple users struggle with the same feature, it is more likely a UX problem rather than just a learning curve issue. For example, while exploring Focus Bear, 
I imagined a usability test where several users had difficulty finding how to start a focus session. Since different users experienced the same problem, I would 
consider improving the visibility or labeling of the feature rather than assuming users simply needed more time to learn it.

## How can UX designers ensure they don’t influence user responses during an interview?
UX designers should ask neutral questions and avoid leading users toward specific answers. For example, instead of asking, “Do you like this feature?”, I would 
ask, “Can you walk me through how you used this feature?” or “What were you expecting to happen here?” These questions encourage users to explain their experience 
honestly without being influenced.

## What’s the best way to document usability testing results so they are useful for design decisions?
The best way is to organize findings by common issues, user quotes, and observed behaviors. I would record notes in a spreadsheet or Figma and group similar 
problems together. This type of documentation makes it easier to identify patterns and prioritize improvements.
