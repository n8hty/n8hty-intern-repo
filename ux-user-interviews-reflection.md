# How to Conduct User Interviews & Usability Testing

## If a user struggles with a feature during testing, how do you know if it’s a UX problem or a learning curve issue?
If multiple users struggle with the same feature, it is more likely a UX problem rather than just a learning curve issue. 
In Focus Bear, if several users cannot easily find the focus timer or understand how to start a session, the design may need to be clearer.

## How can UX designers ensure they don’t influence user responses during an interview?
UX designers should ask neutral questions and avoid leading users toward specific answers. 
Instead of asking “Do you like this feature?”, it is better to ask “How did this feature feel to use?” to encourage honest feedback.

## What’s the best way to document usability testing results so they are useful for design decisions?
The best way is to organize findings by common issues, user quotes, and observed behaviors. Using screenshots, notes, and categorized feedback in tools like Figma 
or spreadsheets also makes it easier for teams to identify patterns and prioritize improvements.
