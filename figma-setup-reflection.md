# Get Familiar with Figma for UI/UX Design

## What are the core features of Figma, and how is it different from other design tools?
The core features of figma include creating interfaces, designing prototypes, wireframes, and real-time collaboration and commenting.
It is different compared to other tools because it is browser-based, stores projects in the cloud, and focuses heavily on real-time collaboration. 

## How does Figma support collaboration between designers, developers, and PMs?
Figma supports collaboration by allowing multiple users to work on the same project in real time. It also provides developer tools such as measurements and code 
inspection, making communications faster and organized.

## What are components, variants, and auto-layout, and how do they improve design efficiency?
The following are features of Figma that help improve design efficiency, consistency, and reduces repetitive work:

### Components
These are reusable elements that help maintain consistency throughout the project. 

### Variants
These are different versions of the component organized in a single component set.

### Auto-layout
This is a customizable feature that automatically arranges elements.

## How do you share and hand off designs to developers using Figma?
Firstly, I would share my designs with developers using the Figma prototype link so they can understand the flow, interactions, and how each feature works. 
Next, I would use Figma’s inspect and developer handoff features to provide important details such as measurements, typography, colors, spacing, and assets. 
I would also communicate through comments and notes to ensure the developers clearly understand the design and functionality before implementation.

# Reflection

## How would you organize a design system in Figma to ensure consistency?
To ensure consistency in figma, I would use features such as components, variants, and auto-layout.
I would group related components into organized sections and use clear naming conventions so team members can easily find and reuse assets.

## If a developer needs details about a design component, how would you provide that using Figma?
I would provide them the project link and allow them to view the design using the inspect and developer features. This would help them access important details
and I would also use comments or notes when necessary to explain specific interactions, behaviors, or functionality more clearly.

## What challenges might arise when collaborating in Figma, and how can you avoid them?
The challenges when collaborating using figma includes miscommunication, overlapping edits, disorganized and unnamed components.
A way to avoid these problems should be to use organized design systems, clear naming conventions, and regular communication to maintain consistency and
improve collaboration.
