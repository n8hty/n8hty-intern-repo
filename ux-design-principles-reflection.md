# Design Principles at Focus Bear

## What are Focus Bear’s design priorities? (Simplicity, low-friction interactions, accessibility)
Focus Bear focuses on simplicity, low-friction interactions, and accessibility to help users stay focused without feeling overwhelmed. The interface should reduce cognitive load by keeping layouts clean, navigation predictable, and actions easy to understand.

## How should UI elements, colors, and typography support neurodivergent users?
UI elements, colors, and typography should help users focus without overstimulation. For example, softer dark mode colors, clear spacing, readable fonts, and large buttons can make the interface feel calmer and easier to use for neurodivergent users.

## Why does consistency in navigation & interaction patterns matter for usability?
Consistency helps users build familiarity and confidence while using the app. Keeping features like focus sessions, task buttons, and navigation in the same location reduces confusion and helps users stay focused.

## How can small UI details (e.g., button placement, animations, tooltips) improve or harm usability?
Small UI details can improve usability by making actions clearer and easier to learn. However, too many animations, cluttered tooltips, or inconsistent button placement can become distracting and overwhelming, especially for users with AuDHD.

# Reflection

## How can Focus Bear maintain a minimal, distraction-free UI while still offering powerful features?
Focus Bear can stay minimal by prioritizing the most important actions first while hiding advanced settings until needed. I noticed that the focus timer and task
related actions are usually easy to access, which reduces friction when starting a session. I think adding more progressive disclosure, such as collapsible
advanced settings panels, would help keep the interface cleaner for new users while still supporting power users.

## If a design change improves aesthetics but reduces usability, how should UX handle it?
UX should prioritize usability if a visual change makes the product harder to use. For example, adding heavy animations or lower-contrast text may look modern
but could distract neurodivergent users or reduce readability during focus sessions. Personally, I think subtle animations and strong contrast work better for
productivity apps because they support concentration instead of drawing attention away from tasks.

## What are the risks of overloading users with too many choices or settings?
Too many choices can overwhelm users, increase decision fatigue, and make the app feel stressful instead of helpful. For Focus Bear, I think keeping only the
essential options visible by default is important because neurodivergent users may lose focus when presented with too many choices in one interface. 
One thing I noticed is that productivity apps work best when users can immediately understand what to do next without needing to think too much about
configuration.
