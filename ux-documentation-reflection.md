# Learn How to Use Google Docs for UX Documentation

## What types of UX documentation are typically created in Google Docs?
Based on my experience, it would be case studies, user research notes, and product design/planning.

## How should design decisions and rationale be documented for future reference?
Design decisions and rationale should be documented clearly and organized for easy collaboration and future iterations.
Include screenshots, version history, and meeting notes to help the team maintain consistency and track changes.

## What is the best way to structure user research findings to make them easy to read?
By adding and organizing information into clear sections with summaries and key insights. Use bullet/numbered points and visual diagrams
to make it easier to scan and understand.

## How can UX documentation be used to support accessibility and inclusive design?
UX documentation can be used to support accessibility and inclusive design by clearly identifying user needs and usability considerations throughout the design
process. It cna also help teams maintain consistency and ensure that acessibility is considered for future design and devlopment decisions.

# Reflection

## If you conducted a user testing session, how would you document the results?
If I conducted a user testing session, I would document my observations, usability issues, user feedback, and recommendations in separate sections to make 
the findings easier for me to organize and understand. This helps me summarize what users actually need based on their experience while making the results easy for
the team to review for future improvements.

## How can a well-structured UX document help prevent miscommunication between teams?
A well-structured UX document helps prevent miscommunication between teams because it keeps the information simple but precise and clear about the design details,
project goals, user needs, and expected functionality of the project.

## What challenges might arise when writing UX documentation, and how can you avoid them?
Aside from miscommunication, the challenges that might arise when writing UX documentation include excessive information, outdated content, 
and inconsistent formatting. To avoid these problems, documentation should be clear, concise, and regularly updated.
